# 核定義：伝達核v1.0 動物軸版 — ソロ切片仕様（凍結 2026-07-17）

本ファイルはソロ切片プロトタイプ実装の唯一の典拠。実装者（Claude Code）はここに無い設計判断を独自に行わない。不明点・矛盾は実装で解決せず報告する。

作業名：伝達核v1.0（評価軸の秘匿伝達 × 燃焼式生命経済）
位置づけ：ソロ切片＝判定外・ネガティブフィルタ専用。**本切片はこの系譜の最後の構造改訂であり、結果が薄い場合は系譜ごとのkill/keep議論に入る（事前宣言）。**

## 0. v0.1からの変更（要約）

1. 秘密を「順列」から「評価軸1本」へ変更（帳簿づけ→エウレカ構造）
2. 記号→動物カード。全軸の順位表は**公開情報**（UIで常時参照可）。秘密は「どの軸が有効か」のみ
3. アンカーを「軸の消去情報」へ変更
4. 経済を燃焼式へ変更：賭けた命は戻らない。ポットは勝者の**得点**に変換。命と得点は別勘定
5. 時間ドレイン（チェスクロック式酸素）を設定トグルで追加
6. ベット段階 {1,2,3,5}・レイズ上限3回・オールイン自動キャップ・先手交互はv0.1から継承

## 1. 検証範囲の事前宣言（変更禁止）

ソロ切片で検証**する**もの：
1. エウレカ体感：ヒント＋勝敗結果から軸を特定する「一発でわかった」瞬間が生まれるか（v0.1の帳簿づけとの差）
2. 命の重み：燃焼式で賭けが「削れる」感覚になるか。時間ドレインON時の息苦しさが快い緊張か不快かを各速度で比較
3. ベット連動：解読前後でベット行動が変わるか。{1,2,3,5}の使い分け
4. 終盤の出し合い：全解読後の同時出し読み合い（Goofspiel型）に楽しさがあるか
5. テンポ

検証**できない**もの（宣言・4人テスト対象）：ヒント作りの妙味、共謀の快感、漏洩ジレンマ、命の重みの対人成分。

## 3. ソロ切片の構成

- 人間＝チームA対戦者。NPC＝チームB対戦者。両伝達者はスクリプト（§6）
- マッチ開始時、有効軸1本を8軸から一様ランダムに選ぶ（真軸。判定処理と伝達者スクリプトのみが参照）
- **アンカー（軸消去）**：真軸以外の7軸からランダムに、チームAへ2軸・チームBへ2軸（重複なし）を「真軸ではない」情報として配布。人間は自分の消去2軸のみ開始時に見る。NPCも自分の消去2軸のみ参照可
- 命：各チーム初期30。得点：各チーム初期0（命と別勘定）
- 全8ラウンド（動物カード使い切り）

## 4. ラウンド進行

1. **ヒント**（第1〜4ラウンドのみ）：発信者はR1=A伝達者、R2=B伝達者、R3=A、R4=Bの交互。等級はR1遠・R2遠・R3中・R4直。真軸のヒント表から該当等級の未使用文を選び公開する（同文の再提示禁止）。R5以降ヒントなし
2. **アンティ**：両チーム命1をポットへ（燃焼。以後ポットに入った命は命としては戻らない）
3. **提示**：人間・NPCが残り手札から各1枚を伏せて出す
4. **ベット応酬**：先手はラウンドごと交互（R1は人間先手）。先手：チェック or ベット{1,2,3,5}。後手：（チェックに）チェック=即判定 or ベット／（ベットに）コール／レイズ{1,2,3,5}上乗せ／フォールド。レイズは1ラウンド合計3回まで。支払額は支払側の残命で自動キャップ（残命全部投入＝オールインでコール成立）。支払った命はすべてポットへ（燃焼）
5. **判定**：
   - コール成立 or 両者チェック → ショーダウン：真軸の順位表で比較し**勝敗のみ**公開（軸名・順位数値は非公開）。勝者はポット全額を**得点**に変換して獲得
   - フォールド → 相手がポットを得点に変換して獲得
   - 同一動物 → 引き分け。ポットは次ラウンドへ持ち越し
6. **公開**：決着方法に関わらず両者の出した動物を公開し使用済みへ。ショーダウン時のみ「勝った動物＞負けた動物」が公開制約として記録される（フォールド時は勝敗情報なし）
7. **死亡判定**：「アンティ支払い直後」と「精算直後」の2箇所で、命≤0のチームは即死亡＝敗北。ベット中の一時的な0到達は死亡ではない（精算まで生存判定を持ち越す）
8. **決着**：一方死亡→他方の勝ち。同時死亡→得点の多い方の勝ち（同点は引き分け）。8ラウンド終了時に両者生存→得点の多い方の勝ち（同点は引き分け）

## 5. 時間ドレイン（設定トグル）

- 設定：オフ／遅（30秒ごとに命1）／標準（15秒ごとに命1）／速（7秒ごとに命1）。マッチ開始時に選択。既定はオフ
- ドレインは**人間の入力待ちの間のみ**進行（カード選択待ち・ベット行動待ち）。ラウンド間のログ確認・結果表示中は進行しない。NPCの思考は瞬時でありドレイン対象外
- ドレインで失われた命はポットに入らず消滅（純粋な燃焼）
- ドレインによる命≤0も§4-7の死亡判定タイミングで裁定する（入力待ち中に0以下へ達した場合は、その時点で強制的に判定を行い死亡）
- 残り命とドレイン進行はリアルタイムで視認できること（酸素ゲージ的表示）

## 6. 伝達者スクリプトとNPC

### 6-1. 伝達者スクリプト

§4-1の交互・等級スケジュールに従い、真軸のヒント表から該当等級の未使用文を一様ランダムに選ぶ（シード依存）。ヒントは常に真軸のもの＝構造的に真実。

### 6-2. NPC対戦者

- 知識：候補軸集合を保持。初期値＝全8軸−自分の消去2軸。以後、(a)公開ヒントの解釈集合との積集合、(b)公開制約（ショーダウン勝敗）と矛盾する軸の除外、で厳密更新。**人間の手札・人間の消去軸・真軸を直接参照することは禁止**（真軸は判定処理と伝達者スクリプトのみ参照）
- カード選択：候補軸集合の各軸で自カードの順位を平均して期待強さとし、「敵の残動物が等確率で出る」仮定で期待勝率最大の動物を選ぶ。機械的単調を避けるため一定確率でランダム選択を混合（混合率は実装裁量・報告）
- ベット行動：出した動物の推定勝率を確信度とし、{チェック/1/2/3/5}・コール/レイズ/フォールドへ写像。しきい値は実装裁量（採用値と根拠を報告）。ブラフ率0の正直NPCのみ

## 7. UI・技術要件

- 単一HTML・外部依存なし（CDN不可・ネットワークなし・Firebase不使用）。Windowsローカル file:// 起動で完全動作
- 必須表示：自分の残手札／敵の残動物（公開）／**全8軸の順位表（常時参照可の公開リファレンス）**／自分の消去アンカー2軸／ヒントログ（発信者タグ付き）／公開制約ログ／両チームの命・得点・ポット（命と得点は明確に別表示）／ラウンド番号／ベット経過ログ／時間ドレイン設定と酸素ゲージ
- **軸メモ**：8軸それぞれをタップで 無印→×（違う）→○（これだ）→無印 と巡回できる手動メモ。自動推論・自動反映は一切実装しない（v0.1の8×8グリッドは廃止し、軸メモに置換）
- シード表示＋シード指定入力（真軸・アンカー・ヒント選択・NPC乱数まで再現）
- 行動ログ全文のワンクリックコピー
- `confirm()` / `alert()` / `prompt()` 使用禁止。破壊的操作は二段クリック
- `const VER = "v1.0.0"` を定義し画面右下にバッジ常時表示。修正納品でVER必バンプ

## 8. 自動テスト（必須実装・実行）

無人自動対戦（人間席もNPCロジック代行・ドレインはオフ）100マッチ。検査項目：

1. 完走率（エラー0で100/100）
2. **命・得点の勘定則**：任意時点で「両チーム残命＋ポット＋両チーム得点＋消滅分（ドレイン等）の総和」が初期総命60と一致
3. ヒント整合：発行された全ヒントが真軸のヒント表に属し、等級スケジュール・再提示禁止に適合
4. 整合性不変条件：真軸が、いかなる時点のNPC候補軸集合からも排除されていない。かつ発行済み全ヒントの解釈集合に真軸が含まれる
5. 勝敗内訳（A勝／B勝／引き分け・うち死亡決着数）の報告
