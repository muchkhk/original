## 2. 作業開始の絶対条件

完全列挙・合理性NPC・方策比較・市場監査を開始できるのは、以下がすべてPASSの場合だけ。

```text
X0 = PASS
X1 = PASS
B1 = PASS
I1 = PASS
S0 = PASS
```

一つでも欠ける場合：

```text
STOP_PREHUMAN_SIMULATION
```

で停止する。

---

## 3. 例外：人間マイクロテスト器具

B1またはI1を実施するための最小UI／表／単一HTMLは作成してよい。

その場合の制限：

- 完成ゲームにしない
- NPCを合理化しない
- 得点バランスを最適化しない
- テーマ、演出、アカウント、オンライン同期を作らない
- 5～10回の中心決断を操作・記録できるだけにする
- 作成後、実際の選択は人間が行う

---

## 4. 閉鎖済み作業

次は開始しない。

- 固定タイプ・写像・順列を履歴から復元する
- 入札・配分・投資・認証・初期配置から人物の秘密価値や将来意図を読む
- 追随を阻止へ変えるだけの自己拘束型試行3
- 事実、覗き、思考を単独商品として売る
- 情報機構を先に作り、後から目的・資源葛藤を付ける
- DESIRED_SCENEを効用表で再現するための完全列挙

該当時：

```text
STOP_SUPER_LINEAGE_REPEAT
```

---

## 5. Codexの担当範囲

許可：

- 人間テスト器具の最小実装
- PASS後の完全列挙
- 支配戦略・席順・最終手番・役割交代監査
- 死亡対照・生存対照の実装
- JSON・CSV・ログ・再現手順・監査パッケージ
- 正式文書の機械的統合

禁止：

- 新系譜の設計判断
- KILL後の蘇生案
- 人間発言の推測
- OBSERVED_SCENEの捏造
- 閉鎖超系譜の再命名
- ゲート基準の事後緩和
