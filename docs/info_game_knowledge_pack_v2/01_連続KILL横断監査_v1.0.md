# 情報系新作 連続KILL横断監査 v1.0

作成日：2026-07-19  
対象：オリジナルボードゲーム制作における、情報・秘匿・推理・ブラフ・通信制限を中心とした停止／KILL履歴  
目的：個別案の検死ではなく、探索方法そのものが過去の教訓を反映しているかを監査する

---

## 0. 正式結論

```text
LOCAL_LEARNING = CONFIRMED
GLOBAL_SEARCH_PROGRESS = INSUFFICIENT
CURRENT_SUPER_LINEAGE = CLOSED
ATTEMPT_3 = CANCELLED
NEW_IDEA_GENERATION = TEMPORARILY_STOPPED
NEXT_REQUIRED_STEP = SEARCH_POLICY_V2
```

過去案から局所的な教訓は多数得られている。

- 一意解と無差別の両方を検出できるようになった
- 想定シグナルの論理前提を先に疑えるようになった
- 推定精度、行動変更、局所利益、最終勝率を分離できるようになった
- 死亡対照・生存対照、事前登録、完全列挙、早期停止を導入した
- KILL後の数値調整・追加ルールによる延命を止められるようになった

しかし、探索全体では同じ上位因果鎖の近傍を反復している。

> **検死能力は大きく進歩したが、新しい探索空間へ移動する能力は十分に進歩していない。**

直近の試行1・試行2は、親系統名と受信者の応答を変更したものの、どちらも次の上位因果鎖を保持していた。

```text
本人の利益を伴う観察可能な行動
↓
相手の隠れた価値・タイプ・方策・将来意図を推測
↓
受信者が後続行動を変更
↓
読みを最終利益へ換金
```

したがって「自己拘束型共同事業シグナル」「自己拘束型戦略意図シグナル」を別の新系譜と数えず、上記超系譜の継続試行として扱う。

# 1. 監査範囲

本監査では、同一作品内の大きく異なるプロトタイプを別の「停止単位」として数える。

| 区分 | 件数 | 意味 |
|---|---:|---|
| 監査済みKILLまたは正式KILL | 8 | 数値・完全列挙・事前登録等を伴う正式停止 |
| 開発中止・埋葬・手順0停止 | 13 | 人間テスト、論理検査、ソロ切片等による停止 |
| **合計** | **21** | 連続探索を評価する単位 |

「21作品を完成させた」という意味ではない。  
同じ作品内の別土台・別プロトタイプを含むため、**どの因果鎖を何回試したか**を見るための単位である。

## 最重要の集中

21停止単位のうち、C・E・Gの13単位は、表現は違っても次の問題を中心に置いていた。

```text
隠れた状態・価値・方策・意図がある
↓
何らかの情報を観測・購入・解読する
↓
真実に近づく
↓
行動を変えれば面白くなるはず
```

さらにHの3単位も、ゲームの葛藤より情報機構を先に設計していた。

> **少なくとも16/21停止単位で、情報が「既存の面白い葛藤を変形する層」ではなく、「面白さをゼロから作るエンジン」として置かれていた。**

これは作品別の偶然ではなく、探索の癖である。

# 4. 教訓は得られていたか

## 4-1. 得られていた教訓

次の順に、死因の検出深度は確実に上がっている。

```text
情報が判断に使われない
↓
情報で行動は変わるが、局所利益へ変わらない
↓
局所利益はあるが、最終勝率へ残らない
↓
ブラフが可能でも最善にならない
↓
固定方策・閾値表へ収束する
↓
想定シグナルの論理前提自体が成立しない
↓
検査器具・指標自体が壊れていないかを先に検品する
```

この進歩により、KILLまでの開発コストと誤判定は減った。

## 4-2. 得られていなかった教訓

しかし、各KILL後に行った変更は、多くの場合「死んだ矢印を一本だけ交換する」局所探索だった。

```text
固定タイプ → 私的価値
私的価値 → 将来意図
追随 → 阻止
二択 → 配分
可逆 → 不可逆
```

これらは下流の重要な改善ではあるが、上流の前提、

> **相手の隠れた内面を読み、それを後続行動へ換金することを中心体験にする**

を維持していた。

したがって評価は次になる。

```text
LESSON_ACQUISITION = STRONG
LESSON_GENERALIZATION = WEAK
SEARCH_SPACE_UPDATE = FAILED
```

# 5. 現役KILLゲートv1.0が防げなかった理由

v1.0は、存在する案を安く検死する装置としては機能した。  
試行1・2とも、G2で早期停止できている。

しかし、次の欠陥があった。

## 5-1. 負の選別器であり、正の探索器ではない

10個の質問は「死んでいないか」を判定する。  
「なぜこの領域に成功確率があるのか」「何を根拠にここを掘るのか」は要求しない。

そのため、似た案を高精度でKILLし続けられる。

## 5-2. 親系統の粒度が細かすぎた

`自己拘束型共同事業シグナル`と`自己拘束型戦略意図シグナル`を別の親系統候補と扱ったが、上位因果鎖は同じだった。

親系統名を変更すれば、試行番号を事実上リセットできる構造だった。

## 5-3. 「一矢印を交換」を新系譜と認めた

新系譜判定が、因果グラフ全体ではなく変更点の存在だけを見ていた。

追随を阻止へ変えても、

```text
実利行動→内面推定→受信者応答
```

は残っていた。

## 5-4. 場面起点が願望作文を通した

G0の場面は、実際に発生した場面でなく「発生してほしい場面」でもPASSにできた。

試行2では、

> 人物別の読みで阻止先を変え、逆転する

という場面を先に書いたが、完全列挙では人物別読みと固定表の行動変更が不合格だった。

場面は証拠ではなく仮説だった。

## 5-5. G1の全公開葛藤を机上説明だけでPASSにした

『魔女の調合』の成功との差は、全公開なら葛藤を説明できることではない。

> **情報を外した分割統治そのものが、実際に人間の選択として既に機能していたこと**

である。

v1.0は、表を埋めればG1をPASSできた。人間がその二択を本当に候補にする証拠を要求しなかった。

## 5-6. 検証の順序が「モデル→人間」のままだった

完全列挙は早期KILLに有効だったが、人工的な効用表の中で希望する場面を成立させる作業になった。

人間が実際に何を比較するかを確認する前に、モデルの合理性を精密化していた。

# 6. 『魔女の調合』を生存対照にした差分

| 項目 | 連続KILL系 | 魔女の調合 |
|---|---|---|
| 情報を外した土台 | 一意解、無差別、目的不明、または机上仮定 | 分割統治だけで選択と利害が成立 |
| 情報の役割 | 葛藤をゼロから作る | 既存の分割葛藤をどちらへ傾けるか不確実にする |
| 行動 | 発信・観測・購入・推定のために設計 | 自分の坩堝を取る直接利益行動 |
| シグナル | 狙って設計した信号 | 利益行動から副産物として好みが漏れる |
| 成功の証拠 | Oracle差、行動変更率、完全列挙 | 人間が実際に切り、選び、再戦を望んだ |
| 答え合わせ | 数値・隠れ状態の正解 | 終了後に好みと各選択の意味を振り返れる |
| 複雑性 | 情報機構を成立させるために増える | 情報が土台の同じ行動へ重なる |

最も重要な差は次である。

> **魔女の調合は「情報ゲームを作ろう」として情報から始めていない。  
> 先に機能する分割統治があり、秘密の好みが後からその決断を変形した。**

# 7. 正式な超系譜閉鎖

## 7-1. 状態復元・固定写像系

```text
SUPER_LINEAGE_C_STATE_RECONSTRUCTION = CLOSED
```

次を再開しない。

- 固定タイプ、秘密写像、順列、既知方策を履歴から復元する
- 復元精度を人物読みと呼ぶ
- 写像数、タイプ数、履歴長だけを増やす

## 7-2. 利益行動から内面を読む系

```text
SUPER_LINEAGE_E_BEHAVIOR_TO_HIDDEN_MIND = CLOSED
ATTEMPT_3 = CANCELLED
```

次を再開しない。

- 入札、配分、投資、認証、交換、初期配置から静的価値を読む
- 同じ行動から将来意図・協力意思へ読み替える
- 受信者の応答を追随、阻止、転進へ変更するだけ
- 人物プロフィールを増やし、固定表との差を作ろうとする

## 7-3. 純情報市場系

```text
SUPER_LINEAGE_G_STANDALONE_INFORMATION_MARKET = CLOSED
```

事実・思考・覗き自体を単独商品として売買する核を再開しない。

## 7-4. 情報先行プロセス

```text
INFORMATION_FIRST_DESIGN_PROCESS = PROHIBITED
```

情報機構を先に作り、後から目的・資源葛藤・得点を付ける工程を禁止する。

# 8. 再開条件

新しい情報系新作を開始するには、次の順を守る。

```text
情報を外した、完全公開の最小ゲーム
↓
人間が実際に5～10回選ぶ
↓
複数の選択が本当に候補になった証拠
↓
選択と結果の直接因果・後悔
↓
同じ土台へ情報変換を1個だけ追加
↓
同じ人間テストで、どの選択がどう変わったか確認
↓
その後に完全列挙・シミュレーション
```

情報を追加する前の土台が通らなければ、その案は情報系新作候補にならない。
