次の3探索サイクルでは、中心動詞を以下にしてはならない。

# 2. 新しい設計順序

## 旧順序

```text
魅力的な情報機構
↓
発信者・受信者
↓
利益構造
↓
ゲームの目的
```

## 新順序

```text
情報なしで面白い一手
↓
その一手を繰り返す最小土台
↓
人間による選択・後悔の確認
↓
情報がその一手をどう変形するか
↓
必要なら数値監査
```

---

# 3. Phase 0：探索距離ゲート

新案を考える前に、因果鎖を次の5要素で表す。

```text
中心動詞：
隠れている／変化する対象：
情報の発生源：
情報を受けた後の行動：
最終利益への経路：
```

過去の最も近い停止単位と比較する。

## 新系譜と認める最低条件

次の両方を満たす。

1. **中心動詞が異なる**
2. 5要素中3要素以上が因果的に異なる

テーマ、人数、値域、カード名、UI、公開範囲の変更は差分に数えない。

満たさなければ、

```text
STOP_SUPER_LINEAGE_REPEAT
```

で開始禁止とする。

---

# 4. Phase 1：情報OFF土台

情報、秘匿、人物タイプ、秘密価値、ヒント、履歴推定をすべて外す。

その状態で、最小ゲームを作る。

## 必須条件

- プレイヤーが目指すことを一文で言える
- 一手で比較する候補が2～4個ある
- 候補は同じ希少資源・位置・時間・勝ち筋を奪い合う
- 局面で最善が反転する
- 選択結果が次の手へ残る
- 他者の行動で最善が変わる、または他者との競合が直接発生する

## 禁止

- 秘密を隠したことでしか二択が生じない
- 「後で情報を足せば面白くなる」として土台を仮合格にする
- 数値表だけでPASSする

---

# 5. Phase 2：人間による土台マイクロテスト

完全列挙より先に、人間が情報OFF土台を操作する。

## 最小実施

- 2人以上
- 5～10回の中心決断
- 完成ゲームでなく、紙・表・簡易UIでよい
- 各決断後に一言だけ記録する

### 質問

1. 何と何を比べたか
2. 実際に迷ったもう一方は何か
3. 結果を見て変えたい判断はあるか
4. 次も同じ答えになると思うか

## PASS条件

- 少なくとも2種類の選択が、理解不足でなく実際の候補として選ばれる
- 「別の選択なら結果が変わった」と具体的に説明できる
- 5～10回の後にも一意の固定手へ崩れていない
- 参加者がゲームの目的を同じ意味で説明できる

通らなければ情報を追加しない。

---

# 6. Phase 3：情報変換を一つだけ追加

情報は、新しいパズルを追加するためでなく、Phase 2で確認した同じ決断を変形するために使う。

許可される情報の役割例：

- 行動可能な場所・接続先・順序を変える
- 権利、期限、利用可能性を非対称にする
- 公開情報の到着時刻を変える
- 同じ資源の用途をプレイヤーごとに変える
- 将来の選択肢を直接増減させる

次の3サイクルでは禁止：

- 隠れた人物タイプを当てる
- 行動から秘密価値・将来意図を推定する
- 事実を単独で購入する
- 固定写像・順列・方策を復元する

## 必須記述

```text
情報OFFで確認済みの中心決断：
追加した情報変換：
情報によって変更される同じ決断：
情報がなくても残る葛藤：
追加前後の人間の選択差：
```

---

# 7. Phase 4：情報ONマイクロテスト

同じ土台・同じ参加者で、情報変換を一つだけ追加して5～10回選ぶ。

## PASS条件

- 情報OFFと比較して、中心決断の選択分布または理由が変わる
- 新しい帳簿づけ・暗記・確率計算が主作業にならない
- 元の土台の面白さが残る
- 情報の成功・失敗を結果から説明できる
- 情報を無視する固定手が支配しない

ここまで通過して初めて、Codexによる完全列挙・シミュレーションを許可する。

# 8. 場面起点の扱い変更

v1.0の「ゲーム後に語られる一場面」は二種類へ分ける。

## DESIRED_SCENE

設計前に望む場面。仮説であり証拠ではない。

## OBSERVED_SCENE

人間マイクロテストで実際に起き、本人の言葉で記録された場面。

現役ゲートでPASSに使えるのは`OBSERVED_SCENE`だけとする。

望む場面を書けても、観測されるまでは`HOLD`である。

# 9. 超系譜単位の打切り

試行数は作品名・親系統名でなく、Phase 0の5要素ハッシュで管理する。

```text
LINEAGE_HASH =
中心動詞 |
対象 |
情報発生源 |
受信後行動 |
利益経路
```

## 規則

- 同じハッシュまたは3要素以上一致する案は同一超系譜
- 2回連続で同じゲートに落ちたら、3回目を使わず閉鎖できる
- 3回目は「権利」ではなく絶対上限
- 名前変更で試行番号をリセットしない
- 閉鎖後の再訪には、5要素中3要素以上と中心動詞の変更が必要

# 10. 正の選定基準

KILLゲートだけでなく、次のどれか一つを開始理由として必要とする。

1. 情報OFF土台が人間マイクロテストで既に機能した
2. 既存の生存ゲームから、中心決断だけを抽出している
3. デジタル固有の操作が、情報でなく行動可能性を直接変える
4. 既存成功作『魔女の調合』と同じく、情報が既存の葛藤へ重なる

「空白のメカニクス表に見つけた」「新規性が高い」「名場面を想像できる」だけでは開始理由にしない。

# 12. 次の3サイクルの制約

```text
CYCLE_1_TO_3:
  central_verb_not_in:
    - 読む
    - 推測する
    - 解読する
    - 見抜く
    - 当てる
  hidden_player_mind: FORBIDDEN
  standalone_information_purchase: FORBIDDEN
  information_off_human_test: REQUIRED
  observed_scene_before_simulation: REQUIRED
```

この制約は情報ゲームを禁止しない。

情報を、

- 行動権
- 接続
- 時間
- 利用可能性
- 順序
- 共有範囲

として直接ゲーム状態へ作用させる方向へ探索を移すためのものとする。
